# Perspectives

A perspective is a configured set of editors and views tuned for one kind of work. Switch via the icon rail on the left side of the shell.

- **[Workbench](/help/ide/perspectives/workbench)** — the default; project tree + editors.
- **[Database](/help/ide/perspectives/database)** — schemas, tables, views, SQL console.
- **[Git](/help/ide/perspectives/git)** — clone, pull, push, branch, diff, log.
- **[Documents](/help/ide/perspectives/documents)** — content management (CMIS / S3 / SharePoint).
- **[Jobs](/help/ide/perspectives/jobs)** — scheduled job control.
- **[Operations](/help/ide/perspectives/operations)** — runtime introspection (registry, repository, problems, logs, extensions).
- **[Processes](/help/ide/perspectives/processes)** — running BPMN processes and the user inbox.
- **[Messaging](/help/ide/perspectives/messaging)** — embedded ActiveMQ broker, queues, topics, message browser.
- **[Tracing](/help/ide/perspectives/tracing)** — OpenTelemetry-backed traces and metrics.
- **[Security](/help/ide/perspectives/security)** — roles, access rules, client registrations.
- **[Settings](/help/ide/perspectives/settings)** — branding, locale, preferences.
